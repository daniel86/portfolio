Level of Detail & Culling System

One of the key performance challenges in real-time rendering is deciding what to draw and how detailed to draw it. In REGEN, I designed and implemented a complete Level of Detail (LOD) and Culling system that scales from small demos to large, scene-heavy simulations.

To streamline authoring and debugging, I built a custom mesh viewer GUI. This tool loads meshes, runs edge-collapse operations interactively, and generates impostors with fine-tuned parameters before committing the configuration to a scene. It provides immediate visual feedback, making it easy to balance performance and visual fidelity. Several screenshots in the slideshow illustrate this workflow, showing the quad-tree debug view, mesh simplification process, impostor preview, and a final scene where 200k of fish are rendered efficiently.

For procedural primitives, REGEN dynamically generates multiple LOD levels by subdivision. For imported meshes, I implemented a quadric error metric-based edge collapse algorithm that progressively reduces triangle count while preserving shape and normals. Thresholds, penalties, and boundary constraints are fully configurable, producing smooth and visually faithful LOD transitions.

For very distant objects, geometry can be replaced by impostors: camera-facing billboards generated by taking snapshots of the original mesh from multiple viewpoints. These snapshots capture albedo, normals, and depth, so impostors integrate correctly with deferred shading, shadow mapping, and other passes — all while drastically reducing geometry cost.

Rendering efficiency is about more than just LOD — it’s about not rendering what you can’t see. I implemented two high-performance culling paths:

  • CPU Path: Quad-tree spatial index projected onto the YZ plane, SIMD-optimized 2D intersection tests, optional 3D verification, and filling of indirect draw buffers for efficient batching.
  • GPU Path: Compute shader pass performs frustum culling and LOD selection on the GPU, then sorts instances by camera distance using a radix sort and fills indirect draw buffers — resulting in zero-CPU-overhead multi-draw calls.

LOD selection is context-aware: instances pick their LOD based on distance to camera of current rendering context, and impostors automatically replace distant geometry. The result is a robust, scalable rendering pipeline that balances visual quality and performance in real-time.
