openEASE — Cloud-based Robot Experience Platform

openEASE is a cloud platform for storing, querying, and visualising robot experiences (NEEMs). It was developed within the Collaborative Research Center EASE, where researchers across disciplines contributed data and knowledge, making openEASE the central hub for sharing, analyzing, and reproducing experiments. By providing a common semantic data format (NEEMs with the SOMA ontology), the platform enables interdisciplinary integration and collaboration at scale.

Technically, openEASE combines a full-stack infrastructure: a Flask-based backend, a Bootstrap/JavaScript frontend with WebGL visualisations, real-time integration with ROS via Robot Web Tools, and a data science backend based on Hadoop. Data versioning is managed with DVC, and GitLab provides collaborative maintenance of datasets. Together, these components form a reproducible, extensible environment that aligns with the Industry 4.0 vision of data-driven, connected infrastructures.

The platform supports a wide range of use cases: it helps researchers reproduce experiments, demonstrate results to collaborators, and run comparative analyses across datasets. Integration with the KnowRob QA system allows researchers to query episodic memories directly, and the multi-modal UI I developed provides rich answers: events are visualised as 3D renderings, timelines are generated for temporal reasoning, and symbolic explanations are displayed alongside sensor data. This combination dramatically reduces the effort required for knowledge exploration.

The scope of openEASE extends beyond research: it has been used as infrastructure for teaching, where students can interactively explore robot logs and semantic annotations, and it provides an Industry 4.0–style data backbone for recording, storing, and analysing robotic operations. Early demonstrations — such as collaborative robots preparing pizza by combining their experiences with recorded human data — highlight how openEASE can serve as a collective memory for robots, enabling them to answer queries like what happened, why, and how during a task.

Project highlights

  • Designed the NEEM knowledge representation and implemented logic-based query interfaces for reproducible research.
  • Built the full-stack infrastructure: Flask backend, WebGL/Bootstrap frontend, and ROS integration with Robot Web Tools.
  • Integrated a data science stack (Hadoop, DVC, GitLab) for dataset versioning, sharing, and large-scale analysis.
  • Developed the multi-modal UI that visualises answers as 3D events, timelines, and symbolic reasoning outputs.
  • Enabled cross-disciplinary collaboration within the EASE research center, providing a common platform for dataset exchange.
  • Explored applications in teaching and Industry 4.0 data infrastructure for robotics and beyond.

Role: Platform architect & full-stack developer — responsible for knowledge representation, backend/frontend implementation, data science integration, and query/visualisation interface design.
